# Sprint 문서 가이드

이 폴더는 각 Sprint의 상세한 진행 상황을 기록합니다.

---

## 📁 파일 구조

```
sprints/
├── README.md          # 이 파일
├── sprint-0.md        # Sprint 0: 프로젝트 세팅
├── sprint-1.md        # Sprint 1: 캠핑장 상세 및 예약 기초
├── sprint-2.md        # Sprint 2: 결제 및 예약 관리
├── sprint-3.md        # Sprint 3: 네비게이션 및 UX 개선
└── sprint-4.md        # Sprint 4: MVP 테스트 및 최적화
```

---

## 📝 문서 작성 규칙

### 1. 파일명

- 형식: `sprint-{번호}.md`
- 예: `sprint-1.md`, `sprint-2.md`

### 2. 문서 구조

각 Sprint 문서는 다음 섹션을 포함해야 합니다:

```markdown
# Sprint {번호}: {제목}

**상태**: {✅ 완료 | 🔄 진행 중 | ⏳ 대기 중}
**기간**: YYYY-MM-DD ~ YYYY-MM-DD
**목표**: {Sprint의 주요 목표}

---

## 📋 주요 태스크

### 1. {카테고리명}

- [ ] 태스크 1
- [ ] 태스크 2
- [x] 완료된 태스크

**완료도**: XX% (완료/전체)

### 2. {카테고리명}

...

---

**전체 완료**: X/Y 태스크 (약 XX%)
```

████████████░░░░░░░░░ XX%

```

---

## 🎯 완료 기준

- [x] 완료된 기준
```

### 3. 진행도 표시

체크박스를 사용하여 태스크 완료 여부 표시:

- `[ ]` - 미완료
- `[x]` - 완료

진행률 바 사용:

```
████████████████████░░░░░ 80%
████████████░░░░░░░░░░░░░ 50%
████████░░░░░░░░░░░░░░░░░ 30%
```

### 4. 상태 아이콘

- ✅ 완료
- 🔄 진행 중
- ⏳ 대기 중
- ⚠️ 주의 필요
- 🐛 버그/이슈
- 📌 액션 아이템

### 5. 업데이트 주기

- **일일 업데이트**: 작업 완료 시마다 체크박스 업데이트
- **주간 업데이트**: 매주 말 회고 및 통계 업데이트
- **Sprint 종료 시**: 최종 회고 및 완료 처리
