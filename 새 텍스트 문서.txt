1. 프로젝트 구조 규칙
Spring Boot 3.5.6 + Java 21 사용
패키지 구조: com.campstation.camp.{layer}.{domain}
계층별 분리: Entity → DTO → Repository → Service → Controller 순서
2. Java 21 활용 규칙
DTO는 Java 21 Record 사용 (불변 객체)
Compact Constructor로 검증 로직 구현
Virtual Threads 활성화
Pattern Matching 적극 활용
3. Spring 관련 규칙
JPA Auditing 활용 (BaseEntity)
캐싱 전략 적용 (EhCache)
트랜잭션 관리 (@Transactional)
보안 어노테이션 (@PreAuthorize) 활용
4. API 설계 규칙
OpenAPI 3.0 문서화 필수
공통 응답 DTO (ApiResponseDto) 사용
페이징 처리 (PageResponseDto) 표준화
HTTP 상태 코드 정확한 사용
5. 데이터베이스 규칙
JPA + H2 (개발), MySQL (운영)
복합 쿼리는 @Query 사용
인덱스 최적화 고려
Soft Delete 패턴 적용
6. 네이밍 규칙
Entity: 도메인명 (예: User, Campsite, Reservation)
DTO: {Domain}{Type}Dto (예: UserResponseDto, CampsiteCreateRequestDto)
Service: {Domain}Service
Repository: {Domain}Repository
7. 검증 및 예외 처리 규칙
Bean Validation 어노테이션 활용
GlobalExceptionHandler 중앙 집중식 예외 처리
비즈니스 예외 커스텀 정의
8. 로깅 및 모니터링 규칙
SLF4J + Logback 사용
중요 비즈니스 로직 로깅 필수
성능 모니터링 고려
